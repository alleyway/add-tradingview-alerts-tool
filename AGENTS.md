# AGENTS.md

## Overview
This is a TypeScript/Node.js CLI tool for bulk-adding TradingView alerts. It uses Puppeteer to automate the TradingView web interface.

## Build, Test, and Lint Commands

### Build
```bash
npm run build    # TypeScript compilation to ./dist
```

### Test
```bash
npm test                   # Run all tests (vitest run)
npm run test:watch         # Watch mode for development
npm run test:ci            # Run tests with coverage
```

**Run a single test file:**
```bash
npx vitest run src/program.test.ts
```

**Run a single test:**
```bash
npx vitest run -t "test name here"
```

### Lint & Format
```bash
npm run lint    # biome lint --write .
npm run format  # biome format --write .
npm run check   # biome check --write . (lint + format)
npm run ci      # biome ci (lint + format, no auto-fix)
```

### Release
```bash
npm run release      # Stable release (requires ./deploy_master.sh first)
npm run release-beta # Beta release
```

## Code Style

### Biome Configuration
This project uses Biome for linting and formatting (configured in `biome.jsonc`):
- **Indentation:** 2 spaces
- **Line width:** 120 characters
- **Quotes:** Double quotes
- **Semicolons:** As needed
- **Trailing commas:** ES5 style

### Imports
- Use ES module syntax with `.js` extensions in all imports (required for TypeScript ESM)
- Example: `import { fetchSymbolsMain } from "./fetch-symbols.js";`
- Organize imports automatically with Biome

### TypeScript
- **Module type:** ESNext (ES modules)
- **Strict mode:** Disabled
- **Declaration files:** Generated
- **Source maps:** Enabled

### Naming Conventions
- **Classes/Interfaces/Enums:** PascalCase (e.g., `MasterSymbol`, `ISingleAlertSettings`)
- **Functions/Variables:** camelCase (e.g., `fetchSymbolsMain`, `baseDelay`)
- **Constants:** camelCase or SCREAMING_SNAKE_CASE depending on scope
- **Error classes:** Must end with `Error` (e.g., `InvalidSymbolError`, `SelectionError`)
- **Enum members:** UPPER_SNAKE_CASE or PascalCase matching the enum style

### File Organization
- One class per file is preferred
- Group related functionality in the `src/service/` directory
- Test files must be co-located with source and named `*.test.ts`

### Error Handling
- Custom error classes should extend `Error`
- Use `Object.setPrototypeOf(this, ClassName.prototype)` in constructors
- Log errors with appropriate level before re-throwing
- Include context in error messages

### Class Patterns
- Use `implements` for interface compliance
- Public properties are defined inline without explicit visibility modifiers
- Use TypeScript enums for fixed sets of values

## Project Structure
```
src/
  ├── cli.ts          # Entry point
  ├── program.ts      # CLI command definitions
  ├── index.ts        # Main exports
  ├── add-alerts.ts   # Alert creation logic
  ├── fetch-symbols.ts # Symbol fetching logic
  ├── classes.ts      # Custom classes and errors
  ├── interfaces.ts   # TypeScript interfaces
  ├── service/
  │   ├── common-service.ts  # Utilities
  │   ├── exchange-service.ts # Exchange API
  │   ├── log.ts            # Logging
  │   └── tv-page-actions.ts # Puppeteer actions
  └── manifest.json  # Version and metadata
```

## Environment Variables
- `BASE_DELAY` - Base delay in ms (default: 1000)
- `DEBUG` - Enable debug logging ("true" or "1")
- `TEST_TRADINGVIEW_*` - Environment variables for CI tests

## Important Notes
- The `dist/` directory is gitignored and generated by build
- Always run `npm run check` before committing
- Test files use vitest globals (describe, it, expect) - no explicit imports needed
